Aplica metodologías agiles para el desarrollo de software


Materia


Aplica Metodologías Ágiles para el Desarrollo de Software es una asignatura curricular del plan de estudios del CECyTEM, impartida por el docente Jaime Cruz López, cuyo objetivo es desarrollar y fortalecer las habilidades técnicas, analíticas y de resolución de problemas del alumno mediante la aplicación de metodologías ágiles.
La materia se centra en la planificación, organización, gestión y desarrollo de proyectos de software, fomentando el trabajo colaborativo, la comunicación efectiva, la adaptación al cambio y la entrega continua de productos funcionales, utilizando buenas prácticas y metodologías actuales del entorno profesional.


Durante el tercer semestre, esta asignatura tiene como propósito desarrollar en el alumno una comprensión sólida de la diversidad de enfoques, metodologías y soluciones técnicas desde las cuales puede abordarse un mismo problema de desarrollo de software, fomentando el pensamiento lógico, crítico y analítico, así como la toma de decisiones informadas dentro de entornos de trabajo ágiles y colaborativos.
Esto se logra mediante el análisis y la estructuración de código, la comparación de tecnologías y herramientas de desarrollo, así como la construcción de aplicaciones web y programas utilizando HTML, CSS, JavaScript, React y lenguajes orientados a bases de datos como Python, promoviendo soluciones claras, eficientes y bien fundamentadas, alineadas tanto a contextos académicos como a estándares profesionales del sector tecnológico.

Primer parcial - Prácticas

Metodologias y usos


Ciclo de vida de un software

El ciclo de vida del software es el conjunto de etapas ordenadas que guían el desarrollo de un sistema, desde la identificación de una necesidad hasta su mantenimiento y mejora continua.
Su objetivo es organizar el proceso, reducir errores, optimizar recursos y asegurar que el producto final cumpla con los requisitos técnicos y del usuario.


exposición - metodologia de cascada

la metodologia de cascada busca ser rapida, precisa y consisa llevando un proceso rapido y de bajo costo impulsando el desarrollo agil y grupal reparitendose el trabajo con base a las areas de cada uno de los integrantes.


Exposición - metodologia de espiral

La metodología de espiral es un modelo de desarrollo de software que combina la disciplina de los modelos tradicionales con la flexibilidad de la iteración, organizando el proyecto en ciclos repetitivos (espirales).
Cada ciclo integra planificación, análisis de riesgos, desarrollo y evaluación, permitiendo identificar y reducir riesgos de manera temprana antes de avanzar a la siguiente fase.

Exposición - metodologia XP

La metodología XP (Extreme Programming) es una metodología ágil de desarrollo de software enfocada en mejorar la calidad del producto y responder rápidamente a los cambios mediante iteraciones cortas y una comunicación constante entre el equipo de desarrollo y el cliente.
Se basa en prácticas como la programación en pareja, la retroalimentación continua, el desarrollo incremental y la simplicidad en el diseño, priorizando siempre la entrega de valor funcional.

Exposición - metodologia Kanban

La metodología Kanban es un método ágil de gestión del trabajo que se centra en la visualización de tareas, el control del flujo de actividades y la mejora continua.
Su principio clave es limitar el trabajo en progreso, lo que permite detectar cuellos de botella, optimizar tiempos y mantener un ritmo constante de entrega.

Exposición - metodologia scrum(proyecto)

La metodología Scrum es un marco de trabajo ágil orientado a la gestión y desarrollo de proyectos, basado en iteraciones cortas llamadas sprints.
Su objetivo es entregar valor de manera incremental, mejorar la adaptación al cambio y fomentar la colaboración constante entre los integrantes del equipo.

Tipos de software

El software es el conjunto de programas, instrucciones y datos que permiten a un sistema informático funcionar, procesar información y resolver tareas específicas, actuando como el vínculo lógico entre el usuario y el hardware.
Su propósito es automatizar procesos, optimizar recursos y ofrecer soluciones eficientes a necesidades técnicas, administrativas y sociales.


Segundo parcial - Prácticas

Tipos de diagramas y servidores


Diagrama de flujo

Un diagrama de flujo es una representación gráfica y ordenada de un proceso o algoritmo, que muestra paso a paso la secuencia lógica de acciones y decisiones necesarias para resolver un problema.
Utiliza símbolos estandarizados para facilitar la comprensión, el análisis y la comunicación de procedimientos, permitiendo visualizar errores, optimizar procesos y estructurar soluciones de forma clara.

Diagrama de flujo logico

Un diagrama de flujo lógico es una representación gráfica del razonamiento lógico que sigue un proceso para tomar decisiones y ejecutar acciones de manera ordenada.
Se enfoca en mostrar condiciones, alternativas y resultados, utilizando estructuras como secuencias, decisiones y ciclos, sin depender de un lenguaje de programación específico.

Flujo de datos

El flujo de datos es la representación del recorrido que sigue la información dentro de un sistema, mostrando cómo se recibe, procesa, almacena y transforma para generar resultados útiles.
Permite analizar la entrada, salida y tratamiento de datos, identificar dependencias, detectar fallos y optimizar procesos, asegurando que la información llegue correctamente y a tiempo.

G.U.I (interfaz grafica del usuario)

Una G.U.I. (Graphical User Interface) o interfaz gráfica de usuario es el entorno visual que permite a una persona interactuar con un sistema informático de manera intuitiva, mediante ventanas, botones, menús, iconos y formularios, sin necesidad de usar comandos de texto.

servidores

Los servidores son sistemas informáticos diseñados para proporcionar servicios, recursos o información a otros equipos, llamados clientes, a través de una red.
Su función principal es almacenar, procesar y administrar datos, garantizando disponibilidad, seguridad y rendimiento en aplicaciones, sitios web y sistemas empresariales.

scrumban


Scrumban y Jira

Scrumban es una metodología híbrida que combina la estructura de Scrum con la flexibilidad y visualización de Kanban.
Su objetivo es mejorar la gestión del trabajo, mantener un flujo continuo de tareas y permitir una adaptación rápida al cambio, sin perder el control ni la disciplina del equipo.
